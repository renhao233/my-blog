一年前决定学 Rust。原因很简单：连续七年 Stack Overflow 最受喜爱语言，不试试说不过去。

一年后的今天，我可以说：这可能是最难学但也最值得学的语言之一。

## 前三个月：跟编译器打架

刚开始的阶段是最痛苦的。Rust 的编译器就像一个严厉的老师，不放过任何一个潜在的问题。你写的每一行代码，它都要检查：这个变量是谁的？借出去了吗？生命周期够长吗？

最常见的错误是 `borrow checker` 报错。刚开始完全看不懂它在说什么，只能靠 Stack Overflow 和 Rust Book 硬啃。

但熬过这个阶段之后，你会发现一个神奇的事情：当编译器终于放行的时候，你的代码大概率没有 bug。没有空指针、没有数据竞争、没有内存泄漏。这种感觉在其他语言里是体会不到的。

## 六个月：开始享受

大约半年后，borrow checker 不再是我的敌人，而是变成了一个可以信赖的助手。我开始理解 ownership 和 lifetime 的设计哲学，写代码时已经能预判编译器会报什么错。

最喜欢的几个特性：

- **enum + match** — 代数数据类型加上穷举匹配，比任何语言的 switch 都强大
- **Result / Option** — 强制处理错误和空值，再也不用担心忘了 try-catch
- **cargo** — 可能是最好的包管理器之一，fmt、clippy、test 全内置
- **零成本抽象** — 写起来像高级语言，跑起来像 C

## 什么时候该用 Rust

- 需要高性能和内存安全的系统级编程
- CLI 工具——Rust 写命令行工具体验极好
- WebAssembly——Rust 到 WASM 的工具链是最成熟的之一
- 嵌入式、网络服务等对延迟和资源敏感的场景

## 什么时候不该用

- 快速原型——Python/JS 更快出活
- 团队里没人会 Rust——学习曲线太陡，强行推广会拖慢进度
- CRUD 业务系统——杀鸡用牛刀，Java/C#/Go 更合适

---

如果你也想学 Rust，我的建议是：别急着做项目，先把 [The Rust Book](https://doc.rust-lang.org/book/) 从头到尾读一遍，每个例子都亲手敲一遍。前两个月会很痛苦，但坚持下来，你会看到一个不一样的编程世界。
