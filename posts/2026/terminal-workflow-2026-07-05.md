在命令行里待久了，你会逐渐摸索出一套属于自己的工作流。这篇文章分享我目前用下来最顺手的一套配置和习惯。

## 终端复用器：tmux

不管你用不用 tmux，都建议至少了解一下它。简单来说，它让你在一个终端窗口里管理多个会话，即使 SSH 断开了，后台任务也不会丢。

我的 tmux 基本操作：

- `Ctrl+b c` — 新建窗口
- `Ctrl+b ,` — 重命名窗口
- `Ctrl+b %` / `Ctrl+b "` — 垂直/水平分屏
- `Ctrl+b d` — 断开会话（detach），任务继续跑

习惯之后，你会发现自己很少关终端了——永远有一个 tmux 会话在后台挂着，随时 attach 回去接着干活。

## Shell 配置

我用的是 zsh，搭配 `starship` 作为提示符。相比 oh-my-zsh 那一大坨插件，starship 更轻量、更快，而且跨 shell 通用。

几个必配的 alias：

- `alias g='git'`
- `alias k='kubectl'`
- `alias ll='ls -lah'`
- `alias ..='cd ..'`

## 常用工具链

- **fzf** — 模糊搜索一切：文件、历史命令、git 分支、进程
- **ripgrep (rg)** — 比 grep 快一个数量级，默认忽略 .gitignore
- **bat** — 带语法高亮的 cat 替代品
- **fd** — 比 find 更友好的文件搜索
- **jq** — 命令行 JSON 处理神器

这些工具的共同点：更快、更友好、默认行为比传统工具合理。装上之后基本不会再用原来的 grep/find/cat。

## 工作流节奏

我一般这样开启一天的工作：

1. 打开终端，`tmux attach` 回到昨天的会话
2. 在第一个窗口里跑编辑器（vim/neovim）
3. 第二个窗口开一个分屏，左边跑测试/构建，右边跑日志
4. 第三个窗口留给 git 操作和零散命令

这套流程的核心思想是：手不离开键盘，一切操作都在同一个终端里完成。效率提升不是线性的，是跳跃式的——当你习惯了不用鼠标、不用切换窗口，你会发现自己进入了一种"流"的状态。

---

如果你刚开始接触终端，不用一次性配齐所有东西。先装一个 tmux，用一周；再加一个 fzf，感受一下模糊搜索的快感；慢慢来，找到自己最舒服的节奏。
