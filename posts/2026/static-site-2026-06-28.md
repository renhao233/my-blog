我的博客历程大概是这样的：WordPress → Jekyll → Hugo → 手写 HTML。

每一步都在做减法，直到最后减无可减，发现其实挺好的。

## WordPress 时代

入门时谁没用过 WordPress 呢？插件丰富、主题好看、后台管理方便。但问题也很明显：需要 PHP 运行环境、MySQL 数据库、定期更新安全补丁、被垃圾评论轰炸。对于一个只是想写点东西的人来说，运维成本太高了。

## 静态站点生成器

然后转向了 Jekyll 和 Hugo。Markdown 写作，生成纯静态 HTML，部署到 GitHub Pages。看起来很完美。

但坑也不少：

- 模板语法每个生成器都不一样，换一个就要重新学
- 主题定制经常需要深入源码，改着改着就忘了自己本来要写文章
- 依赖版本升级偶尔会 break 构建，修起来很烦
- 折腾主题的时间 > 写文章的时间，本末倒置

## 回到最原始的方式

有一天我突然想通了：我到底需要什么？不就是几个 HTML 文件，能展示文字就行了。

于是一个主页（index.html）、一个 CSS 文件（style.css）、一个文章目录（posts/），搞定。没有构建步骤，没有依赖，不需要 npm install。写完直接浏览器打开，或者 scp 到服务器上。

> 很多时候我们不是在解决问题，而是在为「解决问题的方法」制造问题。

## 好处

- **零依赖** — 不需要任何运行时、构建工具、包管理器
- **极速** — 浏览器直接渲染，比任何 SSG 都快
- **可维护性** — 结构简单到一目了然，十年后也能轻松修改
- **专注内容** — 没有主题配置、插件管理来分散注意力

## 代价

当然也有不方便的地方：没有 RSS 自动生成、没有标签系统、没有搜索。但对我来说，这些都不是必需的。真的需要的时候，写个脚本也不难。
